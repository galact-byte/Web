# 完善检查项必填与操作菜单交互

## Goal

让资产检查项的新增、必填状态与操作入口清晰、可用，避免出现无功能控件或状态来源不明确的情况。

## Confirmed Facts

- `src/context/appReducer.ts` 中手动新增检查项（`ADD_ITEM`）固定创建为 `required: false`，因此新项默认显示为“选填”。
- `ADD_ITEM` 使用 `items: [...a.items, newItem]`，新增检查项当前会追加到该资产检查项列表的最末尾；用户已决定改为插入最前面，便于新增后立即操作。
- 已创建检查项卡片右上角的“必填 / 选填”按钮会触发 `TOGGLE_REQUIRED`，可通过单击直接在两种状态间切换。
- 右上角的竖向三点图标只是静态 `span`，没有点击行为；重命名和删除已经由相邻图标按钮提供。
- 项目没有检查项拖拽排序实现或第三方拖拽依赖；卡片内部已有图片缩略图拖拽排序，不能让整张卡片任意位置都可拖动，以免与图片拖拽、上传及粘贴目标选择冲突。
- 用户需要在触屏电脑上排序，但担心触摸拖拽的误触；采用“普通操作 / 排序”模式切换，仅在排序模式启用检查项拖拽。
- 排序模式只展示排序所需的检查项标题与专用拖拽手柄，隐藏或禁用图片上传、状态切换、重命名、删除及粘贴目标选择等普通操作；退出排序模式后恢复。
- 用户已选定方案 A：普通模式工具栏显示“调整顺序”，进入排序模式后显示明确的“完成排序”按钮和状态说明。
- 用户确认当前在 Web 端验收，但要求排序体验同时覆盖 Electron 客户端；两端复用 React 渲染代码，Electron 通过打包后的 `dist/` 运行。

## Requirements

1. 手动新增检查项默认创建为必填，不额外增加新增前的必填 / 选填选择控件。
2. 检查项卡片通过单击可见的状态控件直接在“必填”和“选填”间切换，切换后立即保存。
3. 手动新增检查项插入当前资产检查项列表最前面，便于新增后立即上传或编辑。
4. 采用方案 A 的上下文模式切换：普通操作模式显示“调整顺序”，排序模式显示状态说明和“完成排序”；默认普通操作模式不启用检查项拖拽。
5. 排序模式支持鼠标和触屏拖拽检查项到任意位置，拖动过程应以实时预览顺序呈现：拖动项以悬浮副本跟随指针，其他项平滑让位；放开后才持久化确认该预览顺序，取消则恢复原顺序。
6. 拖拽只能从清晰的专用拖拽手柄发起，不能占用图片拖拽、上传区、文本输入、状态切换、重命名、删除或粘贴目标选择。
7. 排序模式仅展示排序相关内容；隐藏或禁用图片上传、状态切换、重命名、删除及粘贴目标选择，退出后恢复普通操作。
8. 右上角竖向三点不得作为无功能控件保留：应移除或替换为具备明确且可用用途的控件，且不能与已有动作入口产生无意义重复。
9. 排序拖动时，指针接近检查项滚动容器的顶部或底部边缘，应自动平滑向对应方向滚动；指针离开边缘、松手、取消、退出排序或切换资产时必须立即停止，且不得影响普通操作模式滚动。
10. 不改变普通操作模式下已有图片上传、粘贴目标选择、重命名、删除确认及数据持久化行为。

## Acceptance Criteria

- [ ] 手动新增检查项默认显示为“必填”，无需额外选择步骤。
- [ ] 单击已有检查项的状态控件可在“必填”和“选填”间切换，且状态立即更新并持久化。
- [ ] 新增检查项作为必填项插入当前资产检查项列表的最前面。
- [ ] 默认的普通操作模式不启用检查项拖拽；工具栏的“调整顺序”可进入排序模式，状态说明和“完成排序”可明确退出排序模式。 
- [ ] 排序模式下，用户可用鼠标或触屏通过专用拖拽手柄将检查项移动到任意位置；拖动期间拖动项以悬浮副本跟随，其他项平滑让位并实时呈现预览顺序，放开后顺序立即更新并持久化，取消则恢复原顺序。
- [ ] 排序拖动时，靠近检查项列表顶部/底部边缘会自动平滑滚动；离开边缘、松手或取消后滚动立即停止，且实时预览、悬浮副本与最终排序仍保持正确。
- [ ] Web 与 Electron 客户端均使用同一排序交互实现；Web 浏览器级回归和 Electron 打包构建均通过。
- [ ] 普通操作模式下不会因触屏滑动或点击误触检查项排序；排序模式仅提供标题和拖拽手柄，不暴露图片拖拽、上传、粘贴目标选择、状态切换、重命名或删除等普通操作。
- [ ] 退出排序模式后，原有图片上传、粘贴目标选择、状态切换、重命名和删除操作恢复可用。
- [ ] 右上角不再出现无点击功能的竖向三点。
- [ ] 现有项目中的检查项数据兼容，且项目构建、类型检查与相关验证通过。

## Out of Scope

- 模板管理弹窗中的默认检查项交互改造。
