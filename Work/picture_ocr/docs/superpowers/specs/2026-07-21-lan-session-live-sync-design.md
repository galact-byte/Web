# 局域网采集会话热更新与自适应入口设计

## 目标

让项目工作台中的手机局域网采集会话在电脑端增删或修改资产、检查项后保持有效，手机约每 2 秒自动同步最新可采集结构；关闭二维码弹窗只收起界面而不结束会话，并在工作台工具栏清楚显示“采集中”。项目列表不再提供局域网采集入口，列表布局继续按实际可用宽度自适应。

## 范围与边界

- 仅项目工作台顶部可启动、查看或停止手机局域网采集；项目列表删除该入口及其会话管理代码。
- 不增加“手动布局模式”设置。项目列表依据实际 CSS 容器/视口可用宽度收纳次要信息和操作，核心“打开”保持可见。
- 手机端仅采集图片，不提供资产、分类或检查项编辑能力。
- 手机端保留同一会话 URL 和 token；项目结构改变后不需要重新扫码。
- 上传格式、10MB 限制、随机 token、可信局域网边界、电脑端确认落盘协议保持不变。

## 交互设计

### 工作台会话状态

1. 用户在工作台工具栏点击“手机局域网采集”后打开会话弹窗并启动会话。
2. 会话启动后，工具栏入口显示绿色状态点和“采集中”；点击该按钮可重新打开弹窗查看二维码、复制链接或明确停止会话。
3. 点击弹窗右上角“×”仅收起弹窗，不调用停止会话接口。弹窗中说明“关闭此窗口不会停止手机采集，可通过工具栏的‘采集中’重新打开”。
4. 只有点击“停止会话”、离开项目工作台、关闭桌面程序或会话超时才会停止会话。

### 电脑编辑与手机同步

1. 工作台状态中的分类、资产、检查项、名称、必填状态或图片数量变化后，电脑端将完整最新快照推送给当前会话。
2. 手机端首次加载后每 2 秒请求 `/api/session`；当快照变化时，自动显示新的分类、资产和检查项。
3. 手机当前选中的分类和资产仍存在时保持选择，避免刷新时跳页。
4. 若当前分类或资产被电脑端删除，手机回退至首个可用分类和其首个资产，并显示“电脑端项目结构已更新，当前选择已调整”。
5. 若仅图片数量变化，手机只更新数字，不干扰当前选择或上传状态。

## 技术设计

### 渲染器与桥接层

- `LanBridge` 增加 `updateSession(snapshot)`：Web ZIP 使用受 loopback 和 `X-Evidence-Control` 保护的 `POST /api/control/update`；Electron 使用新的 IPC `lan:update-session`。
- `AppContent` 在会话运行期间监听其 `lanSnapshot` 的结构和图片数量变化，并串行调用 `updateSession`。每次调用传递完整快照，避免增量差异与顺序竞态。
- 调用更新失败时不自动停止会话；工具栏/弹窗显示可恢复错误，后续项目变更仍可再次同步。

### Electron 会话服务

- `electron/lanServer.cjs` 将启动时闭包中的 `snapshot` 和 `allowedItems` 改为可替换的当前会话状态，暴露 `updateSnapshot(nextSnapshot)`。
- `updateSnapshot` 在替换前构建新白名单，再一次性替换快照与白名单；`/api/session` 和 `/api/upload` 始终使用同一份当前状态。
- `electron/main.cjs` 验证调用方、规范化快照，并拒绝 `projectId` 与当前会话项目不一致的更新；成功后调用服务的 `updateSnapshot`。
- 新上传回调读取当前会话项目 ID，而非启动时捕获的旧快照。

### Web ZIP 会话服务

- `start-server.ps1` 新增仅限 loopback 控制端的 `POST /api/control/update`。
- 控制端先使用既有快照规范化逻辑生成新 `snapshot` 和 `allowed`，再在 `SessionLock` 内验证会话存在且 `projectId` 相同，并同时替换两者。
- 手机 token API 继续只允许绑定的私有地址；`/api/session` 和 `/api/upload` 分别读取当前快照与白名单，更新不会改变 token、地址、过期时间、待确认上传或已确认结果。

### 手机轮询

- 将初始快照加载提取为可重入的 `refreshSnapshot`。
- 使用 `setInterval` 每 2000ms 请求最新快照，并在组件卸载时清理。
- 请求失败时，初次加载仍显示连接错误；已加载后的临时失败不清空界面或覆盖正在进行的上传提示，下一轮继续重试。
- 合并快照时基于有效 ID 纠正选择状态，并仅在选择被迫改变时显示更新说明。

## 安全与一致性

- 不因热更新放宽 token、来源地址、控制 API header、白名单、MIME、文件魔数或 10MB 限制。
- 电脑端与服务器均验证 `projectId`，防止页面/调用方将活跃会话指向另一项目。
- 上传请求会按照最新白名单校验：被删除资产或检查项的旧手机页面上传将收到 403；新增项目在刷新后可上传。
- 会话关闭后的更新请求必须失败；不会重新创建会话或恢复失效 token。

## 验证策略

- 扩展 Electron LAN 服务验证：更新快照后 `/api/session` 返回新资产，新增检查项上传通过，旧白名单项目上传被拒绝，关闭后仍失效。
- 扩展 Web ZIP 验证：控制更新仅能从 localhost 且需要 header；同项目更新生效、跨项目更新拒绝、最新白名单生效、停止后不允许更新。
- 扩展前端源级验证：项目列表无局域网入口，桥接层存在更新接口，工作台关闭弹窗不停止会话、工具栏显示“采集中”，手机轮询间隔为 2000ms 且具备选择回退。
- 运行现有 LAN、安全、数据包、构建及 PWA 验证命令；手工烟测真实手机扫码、收起弹窗、电脑新增资产、手机自动出现并上传。

## 非目标

- 不实现服务器推送、WebSocket、跨网络访问、多用户协同或手机端资产编辑。
- 不支持 HEIC。
- 不修改已有项目数据模型或导入导出格式。
