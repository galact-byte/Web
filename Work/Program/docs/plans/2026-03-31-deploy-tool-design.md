# 通用部署器设计文档

## 背景

当前有两类真实部署场景：

1. 像当前项目这样，本地打包 ZIP，上传服务器后解压覆盖，再执行前后端构建与服务重启。
2. 常规仓库直接在服务器上 `git pull`，随后执行若干构建、迁移、重启命令。

目标不是只服务当前项目，而是做一个可复用的桌面部署器，支持多个项目配置和多种部署模式，并且把“执行计划预览”和“实时日志”做清楚，降低误操作风险。

## 目标

- 使用 `PyQt6 + Paramiko` 实现本地桌面 GUI 部署器。
- 支持多个项目配置保存与切换。
- 支持三种部署模式：
  - ZIP 上传部署
  - Git 拉取部署
  - 自定义命令部署
- 密码每次手动输入，不落盘。
- 执行前必须先生成并查看“执行计划预览”。
- 执行过程中显示清晰的步骤状态和日志。

## 非目标

- 第一版不做 Windows 远端主机支持，只支持 Linux 远端。
- 第一版不做本地凭据加密保存。
- 第一版不做复杂任务编排器，不支持任意拖拽式流程图。
- 第一版不打包成安装器，先保证源码直接运行可用。

## 交互设计

单窗口三栏布局：

- 左栏：项目列表
  - 新增项目
  - 复制项目
  - 删除项目
  - 设置默认项目
- 中栏：项目配置
  - 项目基础信息
  - 连接信息
  - 模式相关字段
  - 可选部署步骤开关
- 右栏：计划预览 + 执行日志
  - 上方显示即将执行的步骤
  - 下方显示实时日志

底部操作按钮：

- 保存配置
- 测试连接
- 生成计划
- 开始部署
- 停止

## 配置模型

使用本地 JSON 保存多个项目配置，密码不保存。

每个项目配置至少包含：

- 项目名称
- 部署模式
- 服务器地址、端口、用户名
- 本地项目目录
- 远端项目目录
- 部署选项

模式附加字段：

- ZIP 上传：
  - 本地源目录
  - 本地 ZIP 输出目录
  - 远端 ZIP 上传路径
  - 远端解压临时目录
  - 远端覆盖目标目录
- Git 拉取：
  - 远端仓库目录
  - 分支名
  - 拉取前命令
  - 拉取后命令
- 自定义命令：
  - 本地预处理命令列表
  - 远端命令列表

## 执行模型

统一抽象为步骤列表，每个步骤包含：

- 步骤名称
- 执行侧：local / remote / upload
- 执行内容
- 是否高风险
- 是否默认启用

执行前先根据配置生成计划。

执行时按顺序串行运行：

1. 本地预处理步骤
2. 文件打包步骤（如果是 ZIP）
3. SFTP 上传步骤
4. 远端命令步骤
5. 健康检查步骤

任一步失败立即停止，并在日志中高亮失败点。

## 架构设计

拆分为五层：

1. `models`
   - 配置数据结构
   - 步骤数据结构
2. `services`
   - 配置读写
   - ZIP 打包
   - SSH/SFTP 执行
   - 计划生成
3. `runner`
   - 串行执行步骤
   - 推送日志与状态更新
4. `ui`
   - 主窗口
   - 配置表单
   - 项目列表
   - 日志/计划面板
5. `entry`
   - 启动程序

## 测试策略

优先测试非 GUI 核心逻辑：

- 配置文件读写
- 计划生成
- ZIP 打包路径规则
- 步骤执行器的停止/失败行为

GUI 第一版只做基本手工验证，不做复杂自动化 UI 测试。

## 第一版验收标准

- 能保存并切换多个项目配置。
- 能生成三种模式各自的执行计划。
- 能通过 SSH 测试连接。
- 能在 ZIP 模式下执行一次完整部署。
- 执行日志可见，失败可定位。
- 不保存密码。
